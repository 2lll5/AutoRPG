# 星蝕王冠 AutoRPG

一款會隨時間持續生長的繁體中文劍與魔法網頁 RPG。舞台是王國「艾瑟蘭」：百年一次的星蝕正在甦醒古龍、失落誓約與被王權抹去的真名。

## 故事世界

玩家從「星蝕前夜」開始，第一個選擇便分成三條完全不同的傳說：

- **劍道**：拔出王墓誓劍，面對龍鐘塔、軍城、古龍與王冠血統
- **祕法**：進入活著的星典館，追查預言、群星與世界樹書庫
- **靈契**：替無名女巫米菈守住真名，與精靈、亡魂及魔獸締結盟約

核心故事、人物、地點、聖物、彩蛋與結局皆採純劍與魔法設定，不使用伺服器、原始碼、Cookie 墓穴或其他科技世界觀作為劇情元素。

## 分支架構

- 每個實際抵達的章節固定提供 3 個選擇
- 每個選擇各自建立子章節，不限制全域分支數
- 第 5 層最多可形成 243 條路徑，第 10 層最多可形成 59,049 條路徑
- 未走過的傳說不會提前物質化；相同父節點與選擇碼會重建相同內容
- 不同支線只能透過地圖上真實相鄰的城門、石梯、山道、渡船或祕法通道交會

## 連續性規則

每個動態章節會保存並繼承：

- 目前所在城鎮、城堡、森林、王陵或祕境
- 可使用的實體道路與移動過程
- 同行人物、魔獸與未孵化龍蛋
- 尚未完成的誓言、敵人與魔法異變
- 持有聖物、旗標與英雄傾向

角色若從月銀迴廊搭乘「沒有窗戶的魔法升降籠」下降，下一幕一定會描述下降過程並抵達地下月晶窟，不會突然站在森林窗邊或不相干的房間。

## 每小時更新

GitHub Actions 每小時執行 `scripts/generate-story.mjs`：

- 每次新增一份「命運潮汐」素材層，而非固定建立 3 個共同節點
- 所有現存冒險路徑都能利用新潮汐，各自展開 3 個連貫子章節
- 素材包含新敵人、魔法異變、NPC、聖物、線索、感官痕跡與代價
- 網站直接讀取 `public/story.generated.json`，無需等待 Vercel 重新部署
- 2026-07-31 晚間加入星蝕終幕；每條既有路徑會依最後位置、同伴與誓約產生自己的 3 個結局

## 存檔

Browser Cookie `autorpg_state_v4` 保存精簡狀態：

- 目前命運枝條與完整選擇碼
- 最近經過的章節
- 武勇、奧術、羈絆
- 特殊聖物與誓約印記
- 進入動態傳說時的同伴／物品快照
- 輪迴次數與彩蛋狀態

舊科技世界的 v3 存檔不會帶入新世界，以避免人物、地點與誓約互相污染；玩家會從星蝕前夜重新開始。

## 本機執行

```bash
npm install
npm run dev
```

## 測試命運潮汐

```bash
STORY_NOW=2026-07-15T09:00:00+08:00 npm run story:generate
```

同一小時重複執行不會建立重複潮汐。
