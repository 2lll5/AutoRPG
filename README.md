# 零時迴圈 AutoRPG

一款會隨時間持續生長的繁體中文網頁 RPG。核心故事使用明確 `next` 節點；進入即時故事後，改採延遲展開（lazy expansion），不預先把指數成長的完整樹寫進單一 JSON。

## 分支架構

- 每個實際抵達的故事節點固定提供 3 個選擇
- 每個選擇都會建立自己的子節點，沒有「全站每小時只有 3 個節點」的限制
- 第 5 層最多可形成 243 條路徑，第 10 層最多可形成 59,049 條路徑
- 未走過的分支不會預先物質化；相同父節點與選擇碼會確定性地重建相同故事
- 不同支線仍可透過真實存在的門、樓梯、軌道、渡船或通道交會

## 空間與劇情連續性

每個動態節點保存並繼承：

- 目前位置與場景類型
- 可使用的實體出口
- 上一個選擇採用的移動方式
- 同行角色
- 尚未解決的事件
- 攜帶物品、旗標與精神傾向

例如角色從觀測廊選擇搭乘「無窗維護升降梯」，下一幕一定會先描述下降過程並抵達低溫機房；不會突然出現在窗邊、森林或不相干的密室。

## 每小時更新

GitHub Actions 每小時執行 `scripts/generate-story.mjs`：

- 每次新增一份「世界事件素材層」，不是固定新增 3 個共同節點
- 所有現存路徑都能利用新素材層，各自長出 3 個連貫子節點
- 分支文字由父節點位置、出口、人物與選擇碼確定性生成
- 網站直接讀取 `public/story.generated.json`，不需等待 Vercel 重新部署
- 2026-07-31 晚間加入最終素材層；每條已存在路徑會依最後位置與同行者產生自己的 3 個結局

## 存檔

Browser Cookie `autorpg_state_v3` 保存精簡狀態：

- `currentNodeId` 與選擇碼
- 最近經過的節點
- 勇氣、洞察、共鳴
- 特殊物品與分支旗標
- 進入動態故事時的角色／物品快照
- 重啟次數與彩蛋狀態

節點全文不存入 Cookie，而是依路徑重新生成，避免超過瀏覽器 Cookie 大小限制。

## 本機執行

```bash
npm install
npm run dev
```

## 測試排程素材

```bash
STORY_NOW=2026-07-15T09:00:00+08:00 npm run story:generate
```

同一小時重複執行不會建立重複素材層。
